# 管理方針

これまでの経験上、調べながら処理を作るときには全てのコードが説明可能だと感じられているが、後日読みこむと説明、解析が困難だったりする。特にシェルスクリプトなどでそういった影響が顕著だった。

そこで今回は、長年の管理が難しくなることを避けるため、時間経過したあとも詳細な説明が可能だろうと自信をもてるものの他は、以下の方針でリポジトリを管理する。

人間は何をしたいかを記述する。設定や実装は生成AIによって記述される。

何をしたいかと、それを達成するためのものは一つのファイルに収まっていると管理が容易なため、ファイルの先頭部分に「何をしたいか」をコメント、md形式にて記述する。生成AIはそれを読みとってファイルをまるごと生成する。ただし生成時には「何をしたいか」のコメントの部分は常に残すという強い制約をかける。こうすると、いつでもファイル単位で再生成による最新化が行える想定である。
